Detailed Setup

Dropcalc Settings

The config.php file was becoming unreadable, so a separate database table was created for this purpose. The system will detect the existance of the table, and if it isn't there, it will create it with default options. It is highly recommended that you review these options as soon as possible, and become familiar with them, as they dictate how the dropcalc behaves.

Be warned - that the dropcalc has erred on the balance of giving you power. It is not a nanny system, and relies on your understanding of the various options to dictate its behaviour.

The first option determines whether the user is allowed to register a game account via the dropcalc. If you decide not to use the dropcalcs own registration system, then you need to make sure that whatever system you use, does populate the knightdrop table with at least the name and access level fields, or else the dropcalc will have problems if you are using the security features.

A further note on the registration system is that it can prevent people who are banned, from registering an account, but see the options later on for that.

Allow Guest Logon does what it says. Anyone pressing the button will be allowed to log on to the system with limited privileges and use of the system.

Allow Guest Map Access determines whether guests are able to see the maps and locations of mobs. Obviously, if you are restricting mob location access to your population, you will need to set this to off, otherwise they can just log in as a guest and see where your mobs are!

Translate failed logon to guest - rather than failing a logon with an error message, a failed logon will automatically be logged on as if they had hit the guest button. Obviously, if you don't allow the guest button to be available, then set this to off.

The last option explains itself. Set to off if you don't want guests to see the players in clans. This is separate from the paranoia setting, and if you have set paranoia to high, then set this to off.

The Allow All Users option does not apply to guests. It applies to anyone who is logged in to the system. The dropcalc does allow access to its key features by a time system, but this will override that. It won't change it, though, so if you decide to have an open week where anyone can see the mob locations, then you can set this to yes, and when you unset it, things return to normal.

The Give Any New User Rights area requires a bit of thought. This determines whether any newly registered user, gets unlimited access rights to the features listed. It is useful if yo uadopt teh aproach that you want to give your population the maximum functionality, but you can later widthdraw an acounts privileges as a punishment.

This section was a bit of inspiration that hit me one day. It dictates that access to the dropcalcs special festures, can be determined by in-game events.

As an exmple, I could take the party hat (item ID 8557) and can give the ability for any account who has a character that has that item, to access one of the features of the dropcalc. As well as merely having that item, it can be restricted to that they have to have it in their inventory, or even that they have to be wearing it! It can also be further restricted that they have to be on-line at the time ... imaging forcing a character to be on-line, wearing a party hat, to be able to see mob locations!

The on-line restriction doesn't work with character changing, as the character has to be off-line to be able to change the characters appearance.

This section controls the "Top Players" list. The top 10, 50 or 100 players can be shown, and various columns containing character details, can be hidden from view. Those columns that are shown, however, have a sort ability so that if all the columns were shown, the user would have the ability to sort the list by Pk, PvP or Karma.

The last option in this table determines whether or not GM characters are shown as part of the top players list.

The Emergency Teleport was a method introduced to help players get around the bug that happens now and then. Occasionally, when a player logs on, and then selects their character, the game will suddenly crash and an admin has to go in the database and reset their position. The Emergency Teleport feature allows them to go in to the dropcalc and press the button. It will send their character back, automatically, to the characters starter town. The button appears under the thumbnail map.

The GM appearance setting refers to how GM's appear in the "Who's Online" when they are in the game. The standard is that they appear green, while everyone else appears blue. YOu can set this to blue, so that they appear as normal players, or invisible, so that they don't appear in the list at all. Note, however, that GM's are not affected by this and they will always see colour differences.

Detail restrictions alters the amout of detail that players can see when they use the dropcalc. Low means that practically everything is visible. Medium tightens these up a bit and stops players seeing the details on accounts other than their own, other characters locations in the game. High level restricts the visibility of certain characters atributes, and dissables the clan-item-find feature and recipe access features.

Prevent Cross Clan stops players from seeing the inventories of character not owned by them, but are in the same clans as them. The recipe calculator will still work, however, and will report items held by other clan players.

Maximum recipe depth can be set by the administrator, to limit the depth to which the recipe calculator will recurse items that are needed/required for creating a particular recipe.

Blue and green drop chance levels are the points in percentages, in which the dropcalc will change the chance colour from red, to blue and then green.
