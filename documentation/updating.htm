Updating the system

If the worst comes to the worst, simply copy out your config.php file, redownload the latest files, install them all and simply copy your config.php back in. You should then be up and running with the latest version. Any errors after this could be down to missing columns in the knight database tables, but after a certain version the dropcalc is programmed to try and take care of itself, so the disruption should be minimal.

Custom skins can be a little difficult in the case of the coding. The custom skins are made up of the skin.php file along with the two CSS files. Also, if you have put custom information in to the index.php file, you will have to be careful with this file as well. November 2006 saw some major changes to the dropcalc which ended with heavy modification of how these files work, but it is now believed to be relatively stable, so you should be free to develop and distribute your own skins without fear that the code will be changed in the near future.
