Changing The Skin

The CSS file

The CSS file contains the colour and style information for both the drop calc engine itself, and also the current skin. The two parts are separated by a comment and this can be seen on taking a look at the css file itself. It is not advisable to change anything concerning the engine except for the colours and possibly the font styles and sizes. Everything to do with the skin should be fair game for changing in its entirty.

Where the HTML code resides in the PHP and how it works.

index.php contains the code that triggers the skin. The html that makes up the skin is contained in config.php in four routines ... wrap_start, wrap_start_dummy, wrap_end and wrap_end_popup.

To try and explain it, the dropcalc engine is the filling in a sandwich of wrap_start and wrap_end. Wrap_start is responsible for the majority of what you see. It leaves an open ended table cell, and then finishes. The drop calc engine fills the empty table cell with content and then ends. Wrap_end closes the tables and finishes; there isn't really that much in wrap_end. Wrap_start is also responsible for the side menus. The code to trigger the drop calc engine is all in wrap_main; you need to use the same code, but it is then up to you as to how you present it to the user.

Wrap_main_dummy is what the user sees when they first enter the system and need to log on. The side menu contains only enough to request a user name and password, along with a guest logon (if required) and a link to the FAQ if you want it.

wrap_end_popup deals with the ending code for the popup windows, and it is unlikely that you will want to ajust this code.

The writing in index.php is taking the part of the filling in the drop calc sandwich. Read that and you will have a better idea of how the sandwich works.

Be aware of the extra code in wrap_main, as this code pulls out the details on the number of people on line, and also presents various options depending on the access level of the person who is logged on. Think of re-designing the skin as a two fold process ... first, design the site and code it in to html; then merge that html code in to the echo statements that make up the functions. Possibly the best suggestion I can make is to strip it down to the basics, and then build it back up again.
