Statement of Reliability and Liability

Reliability

Working on my own on this short project (with some very good pointers and help from other people) has meant that the program has to assume various truths about the gameserver, the operating system and the database. I've done what I can to make sure that the system works, and I give my sincere thanks to everyone who has helped to debug the drop calc and make it as reliable as possible.

Wherever possible, I have made the programming dynamic, so that any changes to the tables, like new pets added or new player types, are automatically reflected in the system. The addition of new support features by the datapack, however, like when the day and night spawned mobs were added, will require a reprogramming of the drop calc however, so it can not keep up to date with those kinds of changes until I reprogram it.

The design of the system is to have all your variables and settings in one file ... config.php ... so when a new drop calc is released, you should simply be able to drop it in the structure, drop in your own config.php and away it should go. That is the theory, anyway.

I do not accept any responsibility or liability for damages incurred as a result of the use of these programs. Anyone using them or altering them does so at their own risk. It is your own duty and responsibility to take backups of your database before using any tool on them.
