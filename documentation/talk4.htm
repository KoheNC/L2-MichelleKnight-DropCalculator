GM Talkthrough - Server Tools

The server tools are a collection of various buttons that perform various tasks to do with server maintenance and function. They vary in what they do and for the most part are admin only. Some are also password protected, but the basics are ...

Reboot 1 minute - Reboot 3 minutes - Reboot 10 minutes - Abort Reboot

These control the rebooting of the server. The labels are prety obvious, and they do what they say on the tin. The command is issued to the gameserver by telnet, so after issuing a command, check the console to see if the gameserver has actioned it. This option can be carried out by any Gm with the reboot server access level.

Freeform Telnet

Some telnet commands were not issued as buttons in case of accidental clicking. If you want to use a command like shutdown, then this is where you can issue it to the gameserver. In fact, any telnet command can be issued by this option.

Purge Thread Pools - Extreload all - Extinit All - Extunload All

All to do with gameserver control.

Import Logon Accounts

This is usually only used on initial set up of the drop calc on the logon server. It basically imports all the user names from the accounts table and enters it in to the knightdrop table. Note that the import process blanks all the map and recipe calculator details, so these may need to be set manually.

Check Database - Clean Up Database

These options check the validity of the database structure. Check does a quick check, but takes no action. Clean immediatley cleans up all discrepancies found, and gives a report of the status of the databases after the clean. Basically, any pets without callers are erased, any callers without pets are wiped, characters not linked to accounts are wiped (including inventories, etc.) and similar. If there are any outstanding pet problems, or things of that nature, then resolve them before cleaning up the database, or data deletion will occur. If there is a clan without members, then that will get erased also and its warehouse wiped.

The last thing to mention is that if you are running multiple gameservers (and dropcalcs) off the one logon server, then if an account had a character on server 1 but nothing on server 2, then running this on server 2 would say, "Ah, this account has no characters ... delete ... delete ..." and that would kiss goodbye to the account. That is where safe mode comes in ...

Safe mode means that it won't delete accounts if they don't have characters on them. When this is the case, the top option shows up green instead of white, so you know it won't delete the accounts.

1 Year Dormant Delete - 6 month Dormant Delete - 3 month Dormant Delete

If an acocunt has been dormant for this amount of time, then it will have its account, characters and all items erased from the system. It is up to you to decide your own cleaning policy.

Import Recipes

Recipes are normally stored in a flat file in the gameserver filesystem. This option reads in the flat file and creates a database table from it, so the recipe calculator can look up recipes much more easily. It typically only needs to be done once on the installation of the dropcalc, but it does need to be done on each gameserver that the dropcalc is installed on. Obviously, if you change your recipes file than you will have to re-run this tool, but that should be a very rare occurance.

Clone NPC

This allows you to create a custom NPC from an existing NPC. It does all this serverside, and not client side, so no client patches are needed for the client to see an NPC created this way. It allows you to change the NPC name and title, and will create a new NPC in the id region of 90000. You can declare the NPC to be agro or not, the type and some other very basic details. eg. I used it to quickly clone a Lizardman in to a non agressive L2Npc character that I then attached HTML text to and made a part of a quest. Obviously, after creating a new NPC, the server has to be restarted before the change is seen. If you want to change an already existing NPC, then that means going in to the database manually.

Box OK IPs

A list is produced of the IP's that the dropcalc currently has registered as O.K. to dual box. You have the option of deleting any entry from this list.

Logon Status - Restart Logon - Shitdown Logon

These options deal with the logon server, rather than the gameserver, and will require you to re-enter your account password before they can be used. They do what they say on the tin.

Enchant 5+ , 10+ , etc.

These options produce a list of enchanted items on the server that are not owned by GM characters or accounts. The number is the lower threshold that it will begin listing from. The list will produce a name and account of the owner of the item, and you can click on either to get to the character, or the account details.

Create test character

This option creates an administration account and character which is in violation of the bot banning rules. You can thus use this option to test if the automated banning system is, indeed, working.

List Banned Accounts

A list is produced of the accounts that are banned on the system. The order is in date order of latest logon first.
