Routing - Network basics

How does routing work?

Routing works failry simply. Assume that PC 1 is the gamesserver, and PC 2 is a PC that is playing the game. YOu also have another machine which is in the Internet that wants to connect to the gameserver.

For the machine which is inside the network, this is straightforward. It connects to PC 1 simply by using the "internal address" which is the 192.168.0.2 number.

For the machines that are "outside" the network, however, things get a little more complicated. To start with, the outside machine talks with the address 58.123.65.72 ... or rather, the IP address that your ISP has given you. The router then has to make the decision as to what to do with the information.

The router can't make this decision on its own. You have to tell it the "routes" that you want it to take. Each type of traffic has a "port number" associated with it. Classic numbers are 80 for http and 21 for ftp. Your gameserver is probably talking on port 7777 and your logon server on port 2106. Perhaps less known is the MySQL port, which is 3306. You don't have to worry about routing the traffic from inside back to outside again; the router keeps a note of what traffic it has sent where, and it will be able to send the response back to where it came from.

So what do you do with these port numbers? You tell the router that a particular port number is to be directed to an internal number. If, for example, someone from outside wanted to play the game on your internal gameserver, then in the example shown, the router would have to be told to redirect ports 7777 and 2106, to IP address 192.168.0.2

If you had something a little more complicated, however, and your web server was outside the network, then you would have to open a few more ports ... you would have to tell the router to open the MySQL port (3306) the gameserver port (7777) the logon server port (2106) and any telnet ports you have defined ... and point them at the internal gameserver address as well, 192.168.0.2 in this example.

To take this one step further, say that PC 1 had the logon server and gameserver, and PC 2 contained the MySQL database. In that case, the MySQL port would have to be redirected to 192.168.0.3 and the other ports would have to be redirected to 192.1680.2

Easy when you know how!
