Detailed Setup

Cross Gameserver Settings

The settings altered here act across all gameservers that are attached to the same dropcalc.

Hide Language If Only One. If the config.php has been configured for only one language, then it is pointless displaying a drop down list for the languages when it only has one option ... so this settings stops it from being shown.

If the dropcalc only serves one gameserver, then it is pointless it displaying a dropdown list when there is only one gameserver listed.

Hide Skins If Only One is similar to the others. If there is only one skin name specified in the config.php then there is no point in displaying a drop down list with only one option.

Allow Change Server After Logon and Allow Change Skin After Logon - these two apply only to regular users. GM's will always be allowed to change skin and gameserver after logging on, but this option stops regular users from hopping between gameservers or skins once they have logged on ... they must chose their gameserver or skin before they log on, and if they want to change, then they need to log off again. It helps some people maintain focus on which gameserver they are using.

Forcing users to have the default skin is where you can have a specific skin for a specific gameserver, and the users are stuck with that skin. This doesn't apply to GM's.

Showing the servers online status has three settings ... Off to everyone ... Front page for everyone and after logon for GM's ... shows for everyone before and after logon.

The PHP Age is an important setting, especially for Windows servers. Basically, if you are uding PHP 4, then to display the last part of a log file, the dropcalc has to use the "TAIL" command which is available in linux, to get at the last 70 lines of a large config or chat log file. That means Windows users on PHP 4 can not see any of the log files or chat files. PHP 5, however, uses different routines so that both Windows and Linux users can see the last few lines of a large config or chat file easily. PHP 5 is more efficient at this than PHP 4, so if you are using PHP 5, make sure that this is set and you will get better performance from viewing your logs.

PHP 4 users ... read the "Chat System" section of the documentation for more information.

The source e-mail is the email address which the dropcalc determines as the "from" address when sending an e-mail. Some gateways, especially PHP mail(), rely heaivly on this being linked to the correct domain before it will let mail through.

Allow Password Reminder activates an option in the login skin. That option allows someone to enter their e-mail address, or their login name for the game, and have the dropcalc send them a reminder. To help protect against spamming, the system will not allow a repeat mail within 5 minutes of sending one.

Force E-mail authentication. On registering an account with the dropcalc, the system will send an authentication e-mail, to which the user must respond before they will be allowed to play the game.

PHP mail() or SMTP - Most people will chose to use SMTP mail where possible, as the PHP mail() system relies on the ability to edit the php.ini file, which is something that most people can not do on a rented gameserver.

If you chose SMTP, then at the minimum the SMTP server, the port number and the timeout must be specified. If the SMTP server requires authentication, then the logon details can be given below. Some SMTP servers require the localhost setting (usually the same as the domain as the "source e-mail address") to be given also.
