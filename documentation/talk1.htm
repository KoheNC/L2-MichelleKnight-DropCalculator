GM Talkthrough - Initial Page

Quick hints and tips

  1. If dealing with a large list of pets, shops, players or anything, use your browsers FIND function.

What are the options here?

Not a lot, to be honest. The user will see the basic logon section, and if guest logon and register have been enabled, they will see the buttons for those as well.

The dropcalc comes with a basic newbie guide, a simple FAQ and also a guide to connecting to L2J servers using the standard hostfile editing method. These were included as standard to help relieve some of the burden on GM's and also to help people get the most out of the game. If you encounter someone who is a total newbie, you can refer them to these pages to relieve the load on your GM's. If you are running patches which automatically make the changes, then it is a no-brainer to change the basic html that is in the PHP files that deliver these pages.

It really is an effort saver. Just direct them to the drop calc. Oh, and also, once logged on, the system will show the number of on-line users in the page title, so as you wander around the drop calc, that number is always visible to you ... makes it useful if you are watching multiple servers as these numbers show up in the tab.

While we're at it, a quick word or two about running multiple browser sessions. The token that appears after logon is tied to the account logon, which means if you logon again on another browser session, the token will change and it will kick out your previous session. The best trick is to right click on a link (the who's online, for example) and tell it to open in a new page.

Multiple tabs can save your skin if you are running an event, so don't underestimate them, or the need to get the hang of this trick.

If you are running multiple gameservers and have different drop calcs for the different servers, then this is where you need to box clever, as logging on to a different gameservers dropcalc will, again, kill the previous sessions token. (because everything is running from the same logon server) You need to use the same trick as above, but slightly differently. Say we've logged on to http://mygamesite/Lithra/index.php and have authenticated. Right click on the Whos Online and start a new tab or window for that server. Now literally change the server name in the address bar, to one of the other servers dropcalcs ... http://mygamesite/Seighardt/index.php ... leave whatever user and token name information you have, intact. The second session is working off your current logon, because all the servers are using the same logon server.

When in one dropcalc window, you hit the logoff button, it wipes the token ID in the logon server, so any other browser sessions you have open, will then fail.

Practice, practice, practice.
