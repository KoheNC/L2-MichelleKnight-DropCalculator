Telnet System

How do I set this up?

To start with, you have to enable telnet in the gameserver. The telnet configuration is usually found somewhere in the gameserver/config directory. Exactly where depends on the version of the gameserver that you are running. You need to give the system a port number (not a port that is currently being used for a service already running on your system) and a password. This is the information that you then feed in to the config.php file as follows ...

$telnet_host = "localhost";
$telnet_port = "999";
$telnet_password = "password";
$telnet_timeout = 10;

// Login Telnet Settings
$log_telnet_host = "localhost";
$log_telnet_port = "9989";
$log_telnet_timeout = 10;

You will notice two other variables in the config.php file. The first is the telnet_host ip address. If the drop calc is running on the same machine as the gameserver, then this can be left as localhost. If not, then you will need to put in the IP address or a resolvable hostname. The last figure is the telnet timeout in seconds. Normally, the timeout is fine at 10 seconds, but if you have persistant connection problems you might want to raise this number a little higher.

If the gameserver and the drop calc are on the same machine, firewalls shouldn't be an issue. If they are not, and there are firewalls between the machines, then you need to ensure that these are correctly configured. Use a standard telnet client to try and connect to the gameserver first, both on the same machine as the gameserver and on the same machine as the dropcalc; that will help you determine whether the problem is a gameserver telnet issue, or a connection issue between the two machines, or a drop calc problem.

Also note that the logon telnet doesn't have a password. This is because telnet authentication to teh logon server uses the game account details. In use, the dropcalc will automatically use your account name, but it will ask you for your password if you need to use logon server telnet functions.
