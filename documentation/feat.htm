Features

Direct Database Feed

The drop calc feeds directly from the game database. Thus, the figures are live for whatever server it is set up for.

 

Access Control

Varying access levels allow the drop calc to be powerful, but also reasonably secure so that everyday users can find out the information on the drops they require, GM's can kick, ban, chat and all that, while admins can reboot the server or other things like that. The system has been created to allow flexibility in dropcalc roles and security. More information on this in the Initial Setup section.

 

Banning and unbanning players

GM's can ban and kick characters and accounts, but only administrators can ban other GM's and administrators. Accounts which are online at the time of being banned, won't notice the effect until after the account is re-logged. The gameserver won't automatically kick them for you. Also, if a character is online at the time of the character ban, then when the character is de-logged, the gameserver overwrites the ban, so a character has to be kicked before being banned. Note that if an admin bans a GM character, unbanning them will return the character to being a normal player.

 

Adding and deleting items

At the moment, items can be given to a player who is on or off line, but they can only be removed when the player is off line. If they are added with the user on-line, then their inventory must have sufficient free slots to take the items. If off-line when adding, then it doesn't matter. Adding items with the user on line uses the telnet system, and off line is done by database editing. Deleting them is done by removing the items from the database, which has no effect unless the user is off-line. Similar with skills, they can be deleted from the database at any time, but if the player is online at the time, then the gameserver won't take any notice of the database changes until the players character re-logs. This explains why certain options are only available at certain times.

 

Teleporting

Teleporting via the drop calc can only be done when the player is off-line. They can be teleported to either Giran, or the home town for their race. This was included to help out with a bug where a player tries to log on to their character and immediately gets a client crash. It seems to be down to the place they were in when they logged out, so teleporting their character to another location lets them log in again.

 

Chatting

As long as the file system and telnet has been set up correctly, then the system can watch the global server chat, and also individual players chat. The short chat views automatically refresh every five seconds, so you can watch their chat live. The button at the bottom will allow you to chat to the player, but you won't see your admin whisper chat to them appear in the window. Also, things like single quotes come out wrong on their screens, so be careful over what you type.

 

Clan Item Search

Players are able to see the details statistics and inventories of other characters where they have a link via a clan membership. This enables them to track down an ingredient that they need without having to go quest them if an ally already has what they need. It also enables the GM's to track the usage of items and see who has them (and can change/delete them if required) to allow a little more control over cheating, or to assist in events such as relay races, etc.

 

Freight Location

Inventories distinguish in which town freight has been left. The server defaults are that freight stays in the warehouse in which it was left, and I know of few servers that change this feature, so the system will let you know which warehouse the freight was left in. If you happen to see a warehouse titled, "loc x," then it is a new warehouse that the drop calc doesn't know about yet, so look in the newer warehouses.

 

Who's Online

The online list is sorted by character level. GM's get to see another number; that number is the combined level of all the characters linked to that account. This helps identify people who are new to the server, (and can be welcomed aboard or watched) as opposed to long serving players who have just created another character. The on-line list also refreshes itself every 30 seconds.

 

Trusted Players

The system keeps a list of characters that are deemed to be trusted, for example dwarves that won't just laugh and run off with your mats and adena when you want a trade; or other characters who you can request for help in leveling or going after a mob.

 

Upgrading

The drop calc now has the ability to record changes made in the database, when measured against a reference point, so that changes made, such as mob additions and deletions, are more easily portable. It can also backup and restore the key tables, so that it is much easier to save your characters, their pets, items, etc. and then replay them after an upgrade. The system is also flexible enough to look at the tables columns, so that if the datapack project adds more columns, it won't matter so much. (more information in the Database Compare document)
